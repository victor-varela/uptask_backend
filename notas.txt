4- Crear el archivo tsconfig.json (es la configuracion de typescript, Le dice al compilador de TypeScript cómo debe trabajar en tu proyecto. Sin ese archivo, TypeScript no sabe qué reglas aplicar.)

    {
  "compilerOptions": {
    "target": "ES6",
    "module": "commonjs",
    "strict": true,
    "outDir": "./dist"
  }
}

 Esto le dice: "compilá el código a ES6, usá CommonJS, sé estricto, y poné el resultado compilado en la carpeta /dist".

El flujo es asi: Tu código .ts  →  TypeScript lo compila  →  genera .js  →  Node.js ejecuta el .js
 Node.js no entiende TypeScript directamente, por eso necesita ese paso de compilación intermedio.

 5- En server.ts instanciamos express

    Sin Express, con Node.js puro
import http from "http";

const server = http.createServer((req, res) => {
  if (req.url === "/usuarios" && req.method === "GET") {
    res.writeHead(200, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ usuarios: [] }));
  } else if (req.url === "/usuarios" && req.method === "POST") {
    // más código...
  }
  // y así para cada endpoint...
});

Se vuelve un if/else gigante muy rápido. Vos manejás todo a mano: headers, status codes, parsear el body, etc.

Con Express
import express from "express";
const app = express();

app.get("/usuarios", (req, res) => {
  res.json({ usuarios: [] });
});

app.post("/usuarios", (req, res) => {
  // ...
});
Mucho más limpio. Express te da:

Routing — organizar endpoints por método y URL
Middlewares — funciones que se ejecutan antes de llegar al endpoint (autenticación, parsear JSON, logs, etc.)
req y res mejorados — con métodos útiles como res.json(), req.body, req.params

Esta app es un CRM de proyectos por lo que requerimos por cada proyecto cierta informacion (eso es lo que va en el modelo) se puede ir agregando mas informacion al modelo a medida que vamos avanzando. Arrancamos con nombre, cliente y descripcion y luego se puede agregar tareas, archivos, etc. En src cramos la carpeta models y agregamos los modelos.

La arquitectura del proyecto es MVC. Una arquitectura proporciona SEPARACION Y ORDEN, es mas ESCALABLE. Ademas del modelo (datos) contoller (quien consulta y recibe las respuestas del modelo, el que comunica modelo y vista) y la vista (react- Html que muestra los datos) existe el ROUTER (encargado de registrar las URLs o endpoint de la APP) El flujo es asi:

      URL -> controller -> model -> view (json)

Sin el router no hay interaccion porque a partir de la visita a una URL es donde comienza el flujo de la app con el usuario, hasta ahora el flujo era interno de la app, levantar el servidor y conectar la DB.
